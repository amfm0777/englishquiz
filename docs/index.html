Code Review: Riesgos y Mejoras Detectadas

⚠️ Acoplamiento Fuerte con el DOM

La lógica de negocio (puntajes, barajado, llamadas API) está en el mismo archivo que los manipuladores del DOM (document.getElementById, manipulaciones usando `.innerHTML`). Mejora: Separar en arquitectura de servicios o componentes (MVC).

⚠️ Cuello de botella en getAllWords()

Lee el localStorage (`JSON.parse`) y unifica arreglos linealmente cada vez que se requiere la lista. A medida que el usuario guarde cientos de palabras, bloqueará el hilo principal. Mejora: Almacenar el resultado unificado en memoria cache (`let cachedWords = null;`).

🚨 Vulnerabilidad de Seguridad (Admin Login)

La autenticación en local (handleLogin) que se visualiza en `app.js` maneja tokens / contraseñas estáticas en lado del cliente, o simplemente muestra / oculta pantallas CSS. Este modo provee **cero** seguridad real. Mejora: Mover la autenticación a un Backend Real (Node/PHP) usando JWT o Firebase Auth.

Análisis de Mantenibilidad de Testing

Complejidad para Testear: ALTA. Debido a que `app.js` es un monolito acoplado directamente al objeto window, document y APIs web (speechSynthesis), realizar pruebas requiere de un entorno robusto como JSDOM o Cypress/Playwright. Emplear Jest clásico requeriría refactorización masiva para extraer pura lógica.
